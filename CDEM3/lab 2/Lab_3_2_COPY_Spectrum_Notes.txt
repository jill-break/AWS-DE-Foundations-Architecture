PERFORMANCE COMPARISON:
Operation              | Time   | Cost/Month  | Use When
COPY load (100 rows)   | <5s    | FREE        | Daily/hourly analysis
Query COPY data        | <1s    | Included    | Real-time dashboards
Query Spectrum table   | 1-3s   | $6.25/TB    | Rare historical queries
Keep in S3 (Spectrum)  | -      | ~$0.01/mo   | Archive, cold storage

COPY tables:
- Pros: Fast queries (<1s), data in cluster memory
- Cons: Uses cluster storage, requires load step
- Best for: Daily/hourly analysis, joins, dashboards

Spectrum tables:
- Pros: No load time, cheap S3 storage ($0.023/GB/mo)
- Cons: Slower queries (1-3s), $6.25 per TB scanned
- Best for: Rare historical queries, large cold datasets

COST ANALYSIS (actual cluster):
- Cluster: 2 × ra3.xlplus = ~$0.96/hour (~$23/day) — destroy when done
- Data loaded: 100 rows (negligible)
- S3 storage: ~1KB × $0.023/GB = effectively $0/month
- Spectrum query cost: $6.25 per TB scanned (100 rows = $0.00)

DECISION: Use both!
- COPY for hot data (queried daily)
- Spectrum for cold data (queried monthly/yearly)

- Database: spectrum_db (created by Terraform)
- Purpose: Stores metadata for Spectrum external tables

KEY LEARNINGS:
- COPY is the fastest way to bulk-load data into Redshift (parallel, compression-aware)
- Spectrum lets Redshift query S3 directly without loading — zero load time, pay per query
- Glue Data Catalog is the metadata store Spectrum reads from
- Use COPY for hot data (daily queries), Spectrum for cold data (rare historical queries)
- Explicit FORMAT CSV + IGNOREHEADER 1 + MAXERROR prevents silent data errors
- Column named event_timestamp (not timestamp — SQL reserved keyword)
